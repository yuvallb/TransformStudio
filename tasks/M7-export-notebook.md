# M7: Export — Jupyter Notebook

**Goal:** Export pipelines as runnable Jupyter notebooks and polished Python scripts.

**Prerequisites:** M6 complete.

**Estimated effort:** 2x

**README flow:** Flow E

---

## Task 1: Notebook JSON generator

### Implementation

- Create `src/export/notebook.ts`
- Generate nbformat v4 structure:

```typescript
interface Notebook {
 nbformat: 4;
 nbformat_minor: 5;
  metadata: {
    kernelspec: { name: 'python3'; display_name: 'Python 3' };
    language_info: { name: 'python'; version: '3.x' };
  };
  cells: NotebookCell[];
}
```

- Each cell: `markdown` or `code` with `source` as string array (nbformat convention)

---

## Task 2: Markdown cells per node

### Implementation

- For each topo-sorted node, emit markdown cell:

```markdown
## Filter: revenue > 1000

Node ID: `abc123`
Type: `filter`
```

- Use node `label`, optional user `title`, and type ID
- Source nodes: note expected file path placeholder

---

## Task 3: Code cells per node

### Implementation

- One code cell after each markdown cell
- Content = `nodeDefinition.compile(config, inputVars, outputVar, params)`
- Use export variable naming: `df_<nodeId>` or consistent with `codegen.ts`
- First code cell(s) before pipeline: imports

```python
import pandas as pd
import numpy as np

pd.options.mode.copy_on_write = True
```

- If workflow has params, emit params dict cell after imports

---

## Task 5: Polish Python script export

### Implementation

Enhance `src/export/python-script.ts`:

- Header comment block:

```python
# Generated by Transform Studio
# Workflow: My Pipeline
# Created: 2026-06-14T...
```

- Section comments per node matching notebook markdown
- Imports grouped at top
- Params dict if workflow has parameters
- Source nodes: comment that user must adjust file paths

Ensure **single source of truth**: same `compile()` output as execution and code view.

```typescript

expect(nb.nbformat).toBe(4);
expect(nb.cells[0].cell_type).toBe('code'); // imports
```

## Acceptance criteria

### Definition of Done

- [ ] Export notebook → open in Jupyter → run all cells → same logical result as Transform Studio (with user-provided data file at paths).
- [ ] Export menu offers both Python script and Jupyter notebook.
- [ ] Notebook has alternating markdown (node description) + code (compiled snippet) cells.
- [ ] Python script export includes header, imports, section comments, params block.
- [ ] Exported code matches Code View "Full pipeline" content (modulo formatting).
- [ ] README **Flow E** works for both Python script and notebook formats.

### Manual verification

1. Build Filter → GroupBy pipeline with params
2. Export notebook — inspect JSON structure
3. Run in Jupyter with matching CSV — output matches browser preview
4. Export script — compare with Code View panel (should align)
